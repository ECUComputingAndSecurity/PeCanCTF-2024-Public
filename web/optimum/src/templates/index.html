Internal API Documentation

Here at Optimum technologies, we like to keep things simple, and this is reflected in our API documentation. We've made our internal API so easy to use that a child could do it! As this API should never ever be exposed to the internet, no authentication is required to view data, and no super complicated user ID scheme will get in the way of you writing code for it! Our users are numbered, with the first user being 0 and the number after the last user returning a 404 error. Enjoy the documentation!

Endpoints

/api/user/<number>

Parameters: None

Returns: User object


/api/user/<number>/change_name

Parameters: name=<new name>

Returns: Updated user object


/api/user/<number>/change_email

Parameters: email=<new email>

Returns: Updated user object


/api/user/<number>/change_phone

Parameters: phone=<new phone number>

Returns: Updated user object


/api/user/<number>/add_file

Parameters: None, attach file

Returns: Status code (200 success)


/api/user/<number>/remove_file

Parameters: file=<file name>

Returns: Status code (200 success)


Data Formats

All data that contains keys pairs is sent and received in JSON. Files are sent raw as their respective MIME types.


User object

A user object contains the following fields; all strings unless otherwise stated: uid (int), name, email, phone, files (array of file paths)


Error object

An error object returns upon an unsuccessful request and contains the following fields; all strings unless otherwise stated: error
